# D04 — Shareable agent/coin OG cards + share sheet

**Track:** New Features · **Size:** M · **Priority:** P2

## Goal
Every agent, avatar, and coin page gets a beautiful auto-generated social share image (OG card)
and a one-tap share sheet — so anything on the platform is screenshot-and-share ready.

## Why it matters
`CLAUDE.md` quality bar: "What would make someone screenshot this and share it?" Great OG cards
are free distribution; right now links shared to X/Discord likely render plain.

## Context
- Dynamic OG images can be generated by a Vercel function (e.g. `@vercel/og` / satori) per agent/coin, pulling real data (name, avatar render, on-chain badge, key stats).
- On-chain badge is a shared module (memory: `src/shared/onchain-badge.js`) — reflect it on the card.
- Honest data only.

## Scope
- An OG-image endpoint (`api/og/[type].js`) rendering a branded card for agents/avatars/coins from real data; wire `<meta og:image>`/Twitter tags on the respective pages.
- A share sheet component (copy link, X, Discord, download image) on those pages.
- Designed fallback when an entity has no avatar/image yet.
- three.ws branding consistent with Track B; `$THREE` only.

## Definition of done
- Pasting an agent/coin URL into X/Discord/Slack shows a rich, on-brand card with real data; the on-page share sheet works.

## Verify
- Use a social-card validator (or the platform's link preview) on 3 real entity URLs; confirm correct, branded images and working share actions.
