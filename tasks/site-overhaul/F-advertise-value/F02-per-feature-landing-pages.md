# F02 — Per-feature SEO landing pages

**Track:** Advertise & Value · **Size:** M · **Priority:** P2 · **Depends on:** C04, B*

## Goal
A dedicated, SEO-optimized landing page for each flagship capability (selfie→3D scan, text→3D
forge, AR walk, agents/embed, x402 payments, play worlds) — each with a clear value prop, demo,
use-cases, and CTA.

## Why it matters
The platform's value is spread thin across a generic features page. Focused landing pages rank
for intent searches ("turn selfie into 3D avatar," "embed AI avatar on website") and give each
capability a real front door to advertise.

## Context
- Capabilities map to real flows: `/scan`, `/forge`, `/walk`, agents/`/embed`, x402, `/play`.
- Reuse C02 use-cases and F01 demos; Track B components; honest copy/data.

## Scope
- One landing page per flagship feature: H1 with the searched outcome, plain value prop, a live or recorded demo, 2–3 use-cases, social proof (real), and a strong CTA into the flow.
- Proper SEO: titles, meta descriptions, OG tags (tie to D04), structured data, sitemap entries, fast load.
- Internal-link them from the homepage/features so they're discoverable and pass authority.

## Definition of done
- Each flagship feature has a focused, fast, SEO-complete landing page with a working demo and CTA, linked from the site and present in the sitemap.

## Verify
- Lighthouse SEO/performance pass on each; confirm OG tags render a card (D04); CTAs reach the real flow.
