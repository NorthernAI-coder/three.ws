Payment flow

How it works

1. The endpoint answers with a 402 Payment Required challenge (x402 v2).
2. The agent wallet builds and partially signs an SPL USDC TransferChecked — the facilitator co-signs as fee payer, so the agent only signs the transfer.
3. The request retries with the X-PAYMENT header; the facilitator verifies and settles the USDC on Solana mainnet, then the API does the paid work.
