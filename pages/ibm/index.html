The integration.

watsonx.ai and watsonx Orchestrate are first-class providers in the three.ws chat runtime, alongside an open-source MCP server. All grounded in real REST contracts.

POST /api/chat
{ "provider": "watsonx" } streams from watsonx.ai's /ml/v1/text/chat_stream on IBM Granite. IAM bearer minted from WATSONX_API_KEY, scoped to WATSONX_PROJECT_ID.
POST /api/chat
{ "provider": "orchestrate" } calls a watsonx Orchestrate (Agent Connect) OpenAI-compatible /chat/completions endpoint — the 3D avatar becomes the front-end of your enterprise agent.
POST /api/guardian/assess
The Trust Layer — IBM Granite Guardian classifies a message or a proposed autonomous action across jailbreak, harm, bias and more on watsonx.ai, returning an allow / review / block verdict plus a tamper-evident, hash-chained audit record. The same gate runs inline in /api/chat before an avatar sends SOL.
npx @three-ws/ibm-watsonx-mcp
Standalone MCP server for watsonx.ai — five tools against IBM Granite and any model in your account. Works with any MCP host.
# Run a three.ws agent on IBM Granite, in your own watsonx.ai project curl https://three.ws/api/chat \ -H 'content-type: application/json' \ -d '{ "provider": "watsonx", "message": "Introduce yourself." }' # Or expose watsonx.ai to any MCP client with your IBM Cloud credentials WATSONX_API_KEY=... WATSONX_PROJECT_ID=... npx @three-ws/ibm-watsonx-mcp
Open source. The watsonx.ai client is at api/_lib/watsonx.js, the Orchestrate provider at api/_lib/orchestrate.js, the Granite Guardian Trust Layer at api/_lib/granite-guardian.js, and the MCP server at packages/ibm-watsonx-mcp/. Review it, fork it, run your own.
