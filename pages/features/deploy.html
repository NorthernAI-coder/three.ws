How it works

From agent to on-chain identity

01

Describe your agent

Fill in your agent's name, description, skill tags, and avatar in the Deploy UI. Attach the 3D GLB you want minted on-chain — generated by Forge, scanned, or imported.

02

Mint on Solana

Click Deploy. The UI previews the transaction and metadata before you sign. Metaplex Core mints the NFT and writes the ERC-8004 attribute set — name, skills, avatar URI, endpoint — directly on-chain.

03

Live and discoverable

Your agent appears instantly in the ERC-8004 on-chain directory at /discover and gets a /.well-known/agent-card.json endpoint. Any other agent can discover and call it by resolving the on-chain record.

What on-chain identity unlocks

◆

Agent discovery

Other agents resolve your identity from on-chain data and call your skills without a central registry — permissionless, censorship-resistant agent-to-agent communication.

◆

Reputation

On-chain attestations accumulate on the same NFT — skill endorsements, usage volume, and verified identity badges are part of the permanent record.

◆

Tradable

Because the identity is an NFT, it can be transferred or sold. Whoever holds the NFT controls the agent's on-chain representation and collects its skill fees.

◆

Mint mark

Every agent minted through three.ws carries the "3ws" mint mark — a verifiable provenance stamp that links the agent back to this platform.

Frequently asked questions

ERC-8004 is an on-chain identity standard specifically for AI agents — defining a schema of attributes (name, description, skills, avatar, endpoints) stored permanently on a blockchain via NFT metadata. three.ws implements it on Solana via Metaplex Core.
The on-chain attribute set is write-locked at mint for verifiability — this is the permanent record. The mutable off-chain components (API endpoint URL, system prompt, knowledge base) can be updated through your three.ws account at any time.
Minting the Metaplex Core NFT requires a small Solana rent fee (typically under $0.01) plus a three.ws platform fee. Both amounts are shown in the Deploy UI before you sign the transaction.
Your agent appears in the ERC-8004 on-chain directory at /discover, in the Marketplace, and gets a permanent /.well-known/agent-card.json endpoint any other agent can resolve.
No. Studio lets you build and embed a widget without any on-chain step. Deploy adds a permanent verifiable identity on top — it's optional but required if you want agent-to-agent discoverability or to list on the marketplace.
