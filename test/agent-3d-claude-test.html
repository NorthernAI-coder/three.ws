What this tells us.

The viewer is the <agent-3d> custom element from three.ws, loaded as an external module script. The artifact runs in Claude's sandboxed iframe — script-tag loads usually work, but some sandboxes block cross-origin script execution or cross-origin model fetches.

If script load fails, the sandbox is blocking external module scripts entirely. The fallback is to inline the library or use the <iframe src="https://three.ws/a/{chain}/{id}/embed"> approach, which runs in its own browsing context with its own CSP.

If script load succeeds but agent:ready times out, the script ran but the model fetch from assets.three.ws was blocked. Same fallback applies, or host the GLB on a CORS-friendly origin.

To wire in a brain, set brain="claude-sonnet-4-6" and an instructions attribute. That routes chat through their API, which needs the host origin to be allowed in their CORS config.
