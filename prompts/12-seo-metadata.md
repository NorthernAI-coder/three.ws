# 12 · SEO & Metadata Completeness

## Mission
Every page is discoverable, shareable, and indexable: complete titles/descriptions, Open Graph +
Twitter cards, canonical URLs, structured data, sitemap, robots, and rich link previews.

## Context
- Metadata pipeline already exists: `scripts/inject-seo-meta.mjs`, `scripts/inject-blog-seo.mjs`,
  `npm run seo:meta`; page catalog in `data/pages.json`; OG image endpoints under `api/og/`.
- Sitemap + `public/llms.txt` / `llms-full.txt` are generated by the page-index build.

## Tasks
1. **Per-page meta:** confirm every page has a unique, human, keyword-aware `<title>` and
   `<meta description>` (no duplicates, no truncation). Fill gaps via the catalog + injector.
2. **Social cards:** OG + Twitter tags on every page with a working OG image (use/extend the `api/og/`
   generators). Validate with a card-preview check for the top 20 pages.
3. **Canonical + hreflang:** canonical URL on every page; with i18n live (Spanish), correct
   `hreflang` alternates. No canonical pointing at a 404 or non-pretty URL.
4. **Structured data:** add appropriate JSON-LD (Organization, WebSite + SearchAction, Product for
   marketplace items, BreadcrumbList, Article for blog/changelog). Validate against schema.org.
5. **Sitemap + robots:** regenerate sitemap; ensure it lists only canonical, indexable URLs and
   excludes embeds/utility routes; `robots.txt` correct; `llms.txt` accurate.
6. **Crawlability:** confirm key content renders without JS where feasible, or that prerender/meta
   suffices for crawlers; no important page `noindex` by accident.

## Acceptance
- `npm run seo:meta` clean; every page has unique title/description/canonical/OG/Twitter.
- JSON-LD validates; sitemap + robots + llms.txt regenerated and correct.
- Top-20 social previews render correctly (screenshots in `docs/audit/seo-YYYY-MM-DD/`).
