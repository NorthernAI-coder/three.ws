# G1 — Legal pages, content moderation, abuse prevention & takedown

You are a senior engineer + product thinker building **three.ws**. Read `CLAUDE.md`, `STRUCTURE.md`, and `prompts/production-campaign/00b-the-bar.md` first. **Prerequisites:** none (foundational for Track G; coordinate with G4 on token-only styling).

## Why this matters for $1B
A platform that takes money, hosts user prompts, and publishes AI-generated avatars/images cannot be trusted — or sold, or legally operated in the EU/US — without real terms, a privacy policy, and content moderation that actually blocks harmful material. The footer already promises `/legal/privacy` and `/legal/tos` (see `public/nav-data.js` → `DRAWER_LEGAL`), but those pages 404. Every regulator, acquirer, and app store reviewer checks these first. This is the trust floor: legitimate, defensible, safe.

## Current state (read before you write)
- **Legal pages are linked but missing.** `public/nav-data.js` lines ~252–254 link `/legal/privacy` and `/legal/tos`. `ls pages/` shows only `extension-privacy.html`, `extension-terms.html`, `irl-privacy.html` — there is **no platform-wide Privacy Policy or Terms of Use**. The footer links dead-end.
- **Moderation exists but only for anonymous chat.** `api/_lib/moderation.js` is a fail-open NVIDIA NemoGuard content-safety pre-filter wired into `api/chat.js`, `api/chat/proxy.js`, `api/widgets/[id]/[action].js` only. **Forge prompts** (`api/forge.js`, `src/forge-prompt-studio.js`), **generated avatars/images**, and **agent profiles** have no moderation gate. `api/forge.js` mentions provider-side image moderation but runs no first-party text/asset safety pass.
- **Audit log exists.** `api/_lib/audit.js` (`logAudit({ userId, action, resourceId, meta, req })`) writes to the `audit_log` table; `api/audit-log.js` serves it (JSON + CSV) to `/dashboard-next/account`. Migrations under `api/_lib/migrations/2026-05-*-audit-log*.sql`. It is under-instrumented — money/auth/moderation events should all land here.
- **Rate limiting exists.** `api/_lib/rate-limit.js` (`limits`, `rateLimited` via `@upstash/ratelimit`). Audit which abusable endpoints actually apply it.
- **No report/takedown path.** No `api/report*` endpoint, no admin moderation queue. `api/admin/` has revenue/withdrawals/users but no abuse surface.

## Your mission
### 1. Ship the Privacy Policy and Terms of Use pages
Create `pages/legal/privacy.html` and `pages/legal/tos.html` (match the route in `DRAWER_LEGAL`; if the router needs a `pages/legal/` entry in `data/pages.json`, add it). Write real, accurate, plain-language legal copy covering: data collected (wallet addresses, prompts, generated assets, analytics), processors actually used (Solana RPC, OpenAI/Anthropic/NVIDIA worker proxies, Vercel, Upstash, Sentry/Axiom), retention (audit log 365d, generated-asset storage), cookie/local-storage usage, user rights (access/export/deletion — wire to the real audit-log CSV export and any account-delete path), and contact/DPO. Terms must cover acceptable use, prohibited content (the exact categories moderation enforces), IP ownership of generated assets, $THREE is the only token referenced, no investment advice, liability, and DMCA/takedown. Use **design tokens only** (`public/tokens.css`) and the shared nav/footer — these pages must clear the polish bar (TOC, last-updated date, anchor links, all five states are trivially "populated" but typography/spacing must match the platform).

### 2. Extend moderation to Forge prompts, generated assets, and agent profiles
Generalize `api/_lib/moderation.js` (or add a sibling `api/_lib/content-moderation.js` that reuses its NemoGuard call) so every user-generated-content boundary runs a real safety pass: Forge text prompts before generation (`api/forge.js`, `src/forge-prompt-studio.js`), generated **image/avatar** outputs (image-safety classifier on the NVIDIA NIM lane already in use), and agent-profile free-text fields. For signed-in attributable surfaces this can **fail-closed on a confident "unsafe" verdict** (unlike anon chat's fail-open) — refuse with a clear, non-judgmental message and a path forward. Every refusal calls `logAudit({ action: 'moderation.block', ... })`. No mocks — real classifier calls with real timeouts and graceful degradation.

### 3. Build report + takedown
Add `api/report.js` (signed-in or captcha-gated POST: `{ targetType, targetId, reason, detail }`) that writes to a `content_reports` table (new migration in `api/_lib/migrations/`), rate-limited via `api/_lib/rate-limit.js`, audit-logged. Surface a "Report" affordance on user-generated surfaces (Forge gallery items, agent profiles) — accessible button, tokenized styling, confirmation state. Add an admin moderation queue under `api/admin/reports.js` (reuse the existing admin auth pattern in `api/admin/`) that lists open reports and lets an admin resolve/take down, writing the takedown to the audit log and flipping a visibility flag on the target.

### 4. Close abuse / rate-limit gaps and instrument the audit log
Sweep the abusable, money- and compute-touching endpoints (Forge generation, mint/launch, x402 checkout, auth, report) and ensure each applies `rateLimited`/`limits` with sane per-identity budgets. Add `logAudit` calls to the security-relevant events that aren't yet covered: login/logout, withdrawal, mint/launch, moderation block, report filed, takedown. Confirm the existing `/dashboard-next/account` action-log panel renders the new event types correctly (label them human-readably).

### 5. Verify end-to-end
`npm run dev`, walk: footer → privacy/tos render with no console errors; a known-harmful Forge prompt is refused and audit-logged; a report submits and appears in the admin queue; a takedown hides the target and logs. Confirm existing tests pass (`npm test`); add a unit test for the moderation classifier's verdict parsing and the report endpoint's validation/rate-limit boundary.
