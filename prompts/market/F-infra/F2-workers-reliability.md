# F2 — Bulletproof Workers & the Forge Pipeline

You are a senior engineer + product thinker building **three.ws**. Read `CLAUDE.md`, `STRUCTURE.md`, and `prompts/production-campaign/00b-the-bar.md` first. **Prerequisites:** F1 (shares the same DB/Redis/QStash infra) is helpful but not blocking.

## Why this matters for $1B
The workers are where the platform meets real money and real GPUs. `workers/oracle` and `workers/agent-sniper` act on live signals from custodial wallets — a crash mid-trade, a missed heartbeat, or a runaway loop is lost or double-spent funds. The `model-*` generation workers (TRELLIS, Hunyuan3D, TripoSG, TripoSR, text2motion) plus the post-processing chain (remesh, texture, segment, rembg, stylize, unirig) are the Forge product — the first-run "wow." Today that wow already degrades to a designed *"free engines all busy"* / *"provider busy, try again"* state (see `api/forge.js`), which is honest but a dead-ish end. A $1B platform turns backpressure into a queue with a real ETA and a notify-me, never a spinning void or a lost job. This task makes every worker crash-safe, observable, concurrency-bounded, and retry/dead-letter-backed.

## Current state (read before you write)
- **QStash adapter:** `api/_lib/qstash.js` — `qstashEnabled()`, `publishJob({ url, body, delaySeconds, retries=3, deduplicationId })`, `verifyQstashSignature()` (current + next signing key rotation). Used for work exceeding Vercel's exec cap. Gap: no dead-letter capture, no per-job-type retry/backoff policy, thin caller coverage.
- **Long-lived workers (NOT Vercel):** `workers/oracle/` (index.js + score/agent/settle self-scheduling loops, SIGINT/SIGTERM drain, `unhandledRejection` logged; `ORACLE_MODE=simulate|live`, `ORACLE_GLOBAL_KILL`) and `workers/agent-sniper/` (index.js, `heartbeat.js`, `error-tracker.js`, `config.js`, simulate-by-default). Deployed to Cloud Run via `scripts/deploy-sniper.mjs` (minScale=maxScale=1, verifies a fresh heartbeat post-deploy) and `workers/deploy/`.
- **GPU model workers (Cloud Run, Python):** `workers/model-trellis|model-hunyuan3d|model-triposg|model-triposr|model-text2motion/` and `workers/remesh|texture|segment|rembg|stylize|unirig/`. Each is a FastAPI-ish service: `POST /infer → 202 {task_id, status:"queued"}`, `GET /tasks/:id`, `GET /health → {ok, model, gpu_available}`, honors `MAX_CONCURRENT` (default 1), bearer `API_KEY`, `worker_security.py`. Orchestrated by `workers/avatar-pipeline-controller/`.
- **Forge backpressure today:** `api/forge.js` already returns designed busy states (`provider_busy`, "free 3D Spaces are all busy or warming up", "rigger is busy", paid-limit → "free engines stay open"). Tier gating in `api/_lib/forge-tiers.js`. The gap is no durable queue + ETA + notify-me when *every* free engine is busy — the job evaporates and the user must hand-retry.

## Your mission
### 1. Crash-safety + observability on oracle and agent-sniper
Audit the loops in `workers/oracle/` and `workers/agent-sniper/`: every iteration must be wrapped so a single bad signal/trade can't kill the process or wedge the loop; `unhandledRejection`/`uncaughtException` must log to the existing error tracker (sniper has `error-tracker.js`) **and** the platform's Sentry/Axiom path, never just `console`. Confirm graceful SIGTERM drain finishes the in-flight trade (or marks it abandoned safely) before exit. Verify the heartbeat (`heartbeat.js`) is fresh-checked on deploy and surfaced somewhere readable (status page / `/api` health) so a silently-dead worker is detectable. Confirm `*_MODE=simulate` and the global-kill switch genuinely block all real spend.

### 2. Retries, backoff, and a dead-letter path on QStash
Extend `api/_lib/qstash.js` callers (and the receiver endpoints) into a real reliability layer: per-job-type retry counts + exponential backoff (QStash supports this via publish options), idempotent handlers keyed on `deduplicationId` so a re-delivered job never double-acts (double-mint, double-charge), and a **dead-letter capture** — when a job exhausts retries, persist it (Neon table or Redis) with its payload + error so it's inspectable and replayable instead of silently dropped. Every receiver verifies the signature (`verifyQstashSignature`) before doing work. Add a tiny operator path to list/replay dead-lettered jobs.

### 3. Health + concurrency budgets across the model-* and post-process workers
Confirm every GPU worker honors `MAX_CONCURRENT` and rejects (or queues) over-budget `/infer` with a clear status the controller can read — never an OOM/crash. Standardize `GET /health` to report `{ok, model, gpu_available, in_flight, queue_depth}` so the controller and the Forge backpressure logic (mission 4) can route around a saturated engine. Audit `workers/avatar-pipeline-controller/` for: per-stage timeouts, retry on transient 5xx with backoff, and a hard cap so a stuck stage can't pin a job forever. Bearer-auth (`worker_security.py`) and signature checks must be enforced on every worker, not just some.

### 4. Turn "free engines all busy" into a real queue with ETA + notify-me
This is the product win. When `api/forge.js` hits the all-busy state across every free engine, instead of dead-ending: enqueue the job durably (QStash + a Neon `forge_*` jobs row — reuse `forge_seed_jobs`/`forge_creations`-style tables if they fit), return a real position/ETA to the UI, and offer **notify-me** (resolve via the existing notifications path when the GLB is ready). The existing designed busy copy stays as the immediate state, but it now backs onto a queue that actually completes the job and surfaces it — no lost work, no infinite spinner. Wire the populated/queued/failed states honestly (real status polling, no `setTimeout` fake progress). Holders' raised free-lane ceilings (already in `rate-limit.js` via tiered buckets) must reflect in queue priority/ETA consistently.

### 5. Verify it for real
Run the worker(s) locally in `simulate` mode where possible and exercise a job end-to-end; for GPU workers (Cloud Run) verify the `/health` contract and concurrency rejection with a local stub of the contract only if no GPU is reachable — say so explicitly. Drive the Forge all-busy path in a browser against `npm run dev`, confirm the queue/ETA/notify-me states render with real polling and zero console errors, and confirm a queued job actually completes and notifies. Run `npm test` (worker + forge suites).

## Definition of done
Inherits the **global definition of done** in `00-README-orchestration.md`. Plus, mapped to `00b-the-bar.md`:
- **Reliability bar:** no unhandled rejection/exception kills a worker or reaches a user; every money-moving worker action is idempotent and retry-safe; a failed job dead-letters (inspectable + replayable), never silently drops; `simulate`/global-kill provably block real spend.
- **Graceful degradation bar:** the Forge "free engines all busy" state offers a real path forward (queue + ETA + notify-me) that completes the job — exactly the failure the bar names by example. Every worker reports health + concurrency so the system routes around saturation.
- All five Forge states (loading skeleton, empty, error w/ recovery, populated, queued/overflow) designed and reachable; real status polling, no fake progress. `npm test` green; new logic (idempotency, dead-letter, queue) covered.
