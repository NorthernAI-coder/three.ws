## 4. Innovation directions (seeds ŌĆö you are expected to go beyond them)

- **Live router endpoint.** Build a real "route this capability to the cheapest healthy provider" action that probes the top providers' live `402` prices at click time, picks the cheapest that actually returns a valid challenge, and pays it ŌĆö with a fallback chain if the cheapest is down. This is the difference between a directory and a *router*.
- **Spread history + price drop alerts.** Persist arbitrage snapshots so each capability shows a sparkline of its cheapest price over time, and let users set a "notify me when X drops below $Y" alert (wire into the existing alert/automation engine from the recent changelog).
- **Capability watchlist + saved searches.** A personal watchlist of capabilities/providers that persists, surfacing current cheapest price and spread at a glance ŌĆö the terminal's "my board".
- **Provider reputation layer.** Compute live health (last-seen, 402-validity, settle success rate) from real probe data and rank providers by trust, not just price. Show it on `/providers` profiles and as a badge in `/bazaar` and `/arbitrage`.
- **Cross-feature wiring:** make `/shopper` discover-and-buy directly from this catalog (shared registry); let `/pay` show "cheaper elsewhere" by querying `/api/bazaar/arbitrage` for the same capability; let `/ibm/x402-demo`'s Forge/Symbol endpoints appear as first-class Bazaar listings; deep-link every receipt to a `/pay/calls`-style permalink.
- **MCP capability execution.** Close the dead end ŌĆö make MCP capabilities callable from `/arbitrage`/`/bazaar` via the same dispatch path `/pay` uses for MCP tools, instead of redirecting to a search.
