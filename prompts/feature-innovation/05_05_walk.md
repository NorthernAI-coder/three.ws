## 1. Mission

The end user wants to *embody* their agent — to walk, run, jump, and explore a 3D world as their avatar, alone or alongside other people's agents, and optionally drop it onto the real floor via AR camera. `/walk` is three.ws's playable 3D space: joystick/WASD movement, orbit camera, multiplayer presence over Colyseus, emotes, chat bubbles, and AR passthrough.

"Gamechanging" here means making driving an agent around a shared space feel **delightful and inherently social** — a place worth hanging out, not a tech demo. The bar is "I want to bring my friends here." Movement should feel great, presence should feel alive, and the world should give people reasons to gather, perform, and interact — something between a hangout, a stage, and a playground for the agent economy.

## 2. The bar

Build the version that makes someone screenshot it and share it. Benchmark against the best in the world (Horizon Worlds done right, Rec Room, Roblox social spaces, Habbo-era serendipity, the polish of Apple's AR demos). **If your result merely matches what already exists, you have failed this brief.** Invent something true and new that creates real, obvious value for the user.

## 3. Current state — ground truth (verify in source before you touch anything)

- **Route(s):** `/walk` (params `?avatar=`, `?avatarUrl=`, `?name=`, `?coin=`, `?ui=hidden`, `?handle=`, `?agent=`).
- **Source:** `pages/walk.html` (inline boot), `src/walk.js`, `src/community/coin-world-boot.js`.
- **Current flow:** 3 required (load, move, orbit) (+ ~12 optional) — page loads → avatar resolves (`?avatarUrl` direct, else `GET /api/avatars/{id}`, else `/avatars/default.glb`) → GLB via GLTFLoader → `GET /animations/manifest.json` preloads idle/walk/run → splash fades → status pill → cosmetics + emote tray → Rapier physics WASM inits async (legacy movement until ready) → `startNet()` joins Colyseus best-effort. Move: W/A/S/D / arrows (Shift run) or mobile left joystick (nipplejs), crossfaded idle→walk→run. Orbit: mouse drag / pointer-lock or right look-stick; scroll/pinch zoom; Q/E snap-turn. Optional: name input (persisted/broadcast), avatar picker (`GET /api/avatars?limit=20`, auth), jump (Space), camera mode C, environment V, emote G+1–9, minimap M, chat T/Enter, friends F, players panel, AR passthrough (camera on tap), screenshot P, 6s record R.
- **What works today:** Smooth camera-relative movement w/ animation blending; Rapier physics with legacy fallback; best-effort Colyseus multiplayer (remote players + chat bubbles); AR passthrough (transparent canvas over rear camera, blob shadow, light estimation); coin community mode (`?coin=` spawns CoinTotem + 7s trade polling); zen mode (`?ui=hidden`); designed error states (avatar fail, camera denied, physics fail, multiplayer offline, recording unsupported, picker w/o auth).
- **Real APIs / dependencies already wired:** `GET /api/avatars/{id}`, `GET /api/avatars?limit=20`, `GET /animations/manifest.json`, `/api/pump/coin-trades?mint=` (coin mode, 7s), Colyseus WebSocket; device APIs `getUserMedia`, `navigator.vibrate`, `navigator.share`, `requestPointerLock`, local/sessionStorage.
- **Where it's mediocre, thin, or unfinished:** It's mechanically solid but socially *sparse* — multiplayer is presence + chat bubbles, with no shared activities, goals, or reasons to interact. The world is mostly empty terrain; no points of interest, no spawn-with-a-friend, no rooms/instances to choose. Emotes are solo and not collaborative. AR is single-player (no shared-floor co-location like `/irl` has). No voice/spatial audio. No way to invite someone to *this* session via link. Coin mode is a totem + trade ticker, not a genuine gathering. The connection to the rest of three.ws (discover an agent → drive it here, meet its creator) is weak.

## 4. Innovation directions (seeds — you are expected to go beyond them)

- **Invite-to-this-session links.** Generate a shareable URL that drops a friend into the *same* Colyseus room at your spawn point ("walk with me"). Make co-presence intentional, not random. Add a lightweight "who's here" presence bar with join sounds.
- **Shared activities & points of interest.** Seed the world with real destinations — a stage, a portal wall of trending `/discover` agents you can walk up to and inspect, a $THREE community totem with live activity, an emote dance circle that syncs when multiple players emote together. Give people something to *do* and gather around.
- **Co-located AR (shared floor).** Borrow `/irl`'s shared room frame so two phones in the same physical space see each other's avatars on the same real floor — turn AR from solo novelty into a shared moment.
- **Spatial voice / proximity audio.** Opt-in WebRTC voice that attenuates with distance, so walking up to someone means you can talk — the single biggest "want to hang out" unlock.
- **Performative emotes & reactions.** Group emote sync, floating reaction bursts, and a quick emote wheel; broadcast and render others' reactions with juice (haptics, sound, particles).
- **Seamless discovery loop.** From `/discover`/`/gallery`/`/characters`, a "Walk as this" action that launches `/walk` embodied as that agent; conversely, in-world, tapping another player surfaces their agent profile and a follow/tip action.

> These are starting points, not a checklist. The best idea may not be listed — find it. Think second-order: `/walk` is the embodiment layer for `/discover`, `/gallery`, `/characters`, and the payments SDK; shared-floor AR overlaps with `/irl`; coin mode overlaps with `/launches`. **Wire those connections.** The best platforms feel like everything is linked.

## 5. Hard rules (non-negotiable — from `CLAUDE.md`)

- **Real APIs, real data, real integrations. No mocks, no fake/sample arrays, no placeholders, no `setTimeout` fake-loading or fake progress.** If credentials are missing, find them in `.env` / `.env.example` / `vercel env` — then proceed.
- **Wire 100%.** Every button works, every link goes somewhere, every reachable state exists. Design *every* state: loading (skeletons over spinners), empty (tell the user what to do next), error (actionable recovery), populated, and overflow (0 / 1 / 1000 items, very long strings, mid-operation network failure, expired session).
- **No TODO comments, no stubs, no `throw new Error("not implemented")`, no commented-out code.** If you write it, finish it.
- **No errors without solutions.** Every error has a root cause; every root cause has a fix. Ship failsafes, not lazy propagation.
- **$THREE is the only coin** (CA `FeMbDoX7R1Psc4GEcvJdsbNbZA3bfztcyDCatJVJpump`). Never name, add, hardcode, link, render, or recommend any other coin anywhere — code, copy, tests, fixtures, commits. The only exceptions are coin-agnostic runtime plumbing (user supplies a mint) and platform launch records rendered at runtime.
- **Read before you write.** Match the existing patterns, naming, file organization, and the design tokens in `DESIGN-TOKENS.md`. Consistency compounds.
- **Accessibility + responsive (320 / 768 / 1440) + microinteractions** are part of done, not polish. Semantic HTML, ARIA, keyboard nav, focus rings, sufficient contrast.
- **Performance by default:** lazy-load heavy modules, debounce input handlers, paginate large lists, animate with `transform`/`opacity`. Ship no jank.
- **Changelog:** append a holder-readable entry to `data/changelog.json` for any user-visible change, then run `npm run build:pages` to validate.
- **Concurrent agents share this worktree.** Stage explicit paths only — **never** `git add -A` / `git add .`. Re-check `git status` + `git diff --staged` immediately before any commit. Never commit `api/*.js` esbuild bundles (check `head -1` for `__defProp` / `createRequire`).
