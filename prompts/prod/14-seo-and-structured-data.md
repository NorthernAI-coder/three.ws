# 14 — SEO & structured data

> Part of **Road to $1B** (`prompts/road-to-1b/`). Read `00-README.md` and `/CLAUDE.md` first.

**Phase:** 1 — Cross-cutting hardening
**Owns:** `pages/` `<head>`s, `public/sitemap/`, `public/llms.txt`, `public/llms-full.txt`, `data/pages.json`, `scripts/build-page-index.mjs`, `scripts/seo-meta` (`npm run seo:meta`).
**Depends on:** `10` (perf affects ranking). Pairs with `34`.

## Why this matters for $1B
Organic discovery is the cheapest growth channel and a durable moat. 293 pages across
11 sections is a large surface to rank. Done right, search + AI-agent discovery
(llms.txt) compounds for free.

## Mission
Every page is discoverable, correctly titled/described, share-ready, and emits valid
structured data. The sitemap, llms.txt, and AI-discovery surfaces are complete and
current.

## Map
- `npm run seo:meta` audits meta. `npm run build:pages` regenerates page index +
  sitemap + changelog from `data/pages.json` + `data/changelog.json`.
- AI discovery: `public/llms.txt`, `public/llms-full.txt` (regenerated by build).
- x402 endpoint discovery indexing is tracked separately (see memory note) — don't
  duplicate that here, but cross-link.

## Do this
1. Run `npm run seo:meta` and fix every page missing or sharing a `<title>` /
   `<meta description>`. Titles unique, descriptive, < ~60 chars; descriptions
   compelling, < ~155 chars.
2. **Open Graph + Twitter cards** on every page: `og:title/description/image/url/type`
   + `twitter:card` and a real share image (per-section default + per-entity dynamic
   for agents/launches/collections). Verify with a card validator.
3. **Canonical URLs** on every page; resolve duplicate-content (trailing slash, query
   params) via canonical + `vercel.json` redirects.
4. **Structured data (JSON-LD):** `Organization`/`WebSite` site-wide;
   `BreadcrumbList` on nested pages; `Product`/`Offer` on marketplace/skill pages;
   `Article` on blog posts; `SoftwareApplication` on SDK pages. Validate with the Rich
   Results test.
5. **Sitemap:** confirm `public/sitemap/` includes every public page with sane
   priorities and `lastmod`; submit-ready. `robots.txt` correct (allow public, block
   private/API, point to sitemap).
6. **llms.txt / llms-full.txt:** ensure they describe the full product surface and
   stay regenerated by `build:pages`. This is how agents discover the platform — keep
   it first-class.
7. **Semantic + perf:** correct heading hierarchy (shared with prompt `09`), fast LCP
   (prompt `10`), and crawlable content (no JS-only critical content where avoidable).
8. Wire `seo:meta` into CI so a page can't ship without meta.

## Must-not
- Do not duplicate titles/descriptions across pages.
- Do not reference any coin other than `$THREE` in any meta/structured data/share copy.
- Do not block crawlers from public pages or expose private/API routes in the sitemap.

## Acceptance
- [ ] `npm run seo:meta` clean — every page unique title + description.
- [ ] OG/Twitter tags + share images on every page; validated.
- [ ] Canonicals + redirects resolve duplicate content.
- [ ] Valid JSON-LD on org/breadcrumb/product/article/SDK pages (Rich Results passes).
- [ ] Complete sitemap + correct robots.txt.
- [ ] llms.txt/llms-full.txt complete and regenerated by build.
- [ ] `seo:meta` enforced in CI.
