# 17 — ERC-8056 reference: `erc8056` (own the standard)

Read `prompts/robinhood-chain/_shared.md` first. Wave 2 (small, independent — only needs viem
and the chain). Requires core SDK only for cross-checking, not as a dependency.

## Mission
Build `robinhood/erc8056/` — the reference implementation and definitive explainer for
ERC-8056 (`uiMultiplier()`), the corporate-actions standard Robinhood's Stock Tokens use.
It is brand-new, under-documented, and every naive integrator values positions wrong after a
split or dividend. A tiny, perfect package + the canonical explainer site = permanent authority
over the chain's one novel standard. npm `erc8056` (fallbacks: `erc-8056`, `ui-multiplier`).

## Deliverables

1. **Research first, cite everything** — locate the actual ERC/EIP draft text (search
   eips.ethereum.org / ethereum-magicians / Robinhood docs), read a real Stock Token contract's
   verified source on Blockscout, and document the TRUE semantics: multiplier precision/decimals,
   when it changes (splits, reverse splits, dividends?), whether feeds are pre- or
   post-multiplier (the core SDK team found feeds ALREADY multiplier-adjusted — verify
   independently and reconcile; this exact confusion is why the package must exist).
2. **`erc8056` package** (TypeScript, viem peer) —
   - Minimal ABI export + `readUiMultiplier(client, token)` with graceful non-implementer
     detection (`supportsErc8056()`).
   - `trueBalance({ raw, multiplier })`, `trueValue({ raw, multiplier, price, priceIsAdjusted })`
     — the correctness math with EXPLICIT handling of the pre/post-adjusted price question,
     typed so misuse is hard (distinct `AdjustedPrice`/`RawPrice` branded types).
   - `watchMultiplier(client, token, cb)` — detect corporate-action events.
   - Solidity: `IERC8056.sol` interface + a commented reference consumer contract in
     `contracts/` (Foundry project, real tests with fork-mode against a live Stock Token —
     `forge test --fork-url` the public RPC).
3. **The explainer site** (`docs/`, per `_shared.md`) — "Understanding ERC-8056" as the
   canonical resource: what it is, why it exists, the wrong-vs-right worked example with REAL
   on-chain numbers from a real token, live demo widget reading a real multiplier client-side,
   integration recipes (JS + Solidity), FAQ built from the ambiguities you resolved in
   deliverable 1. Write it so it ranks: precise title/meta/headings, self-contained.

## Requirements
- Vitest: math with branded-type misuse cases (must not compile where wrong — use type-level
  tests via `tsd` or expect-type), live read against ≥ 3 real Stock Tokens (paste multipliers).
- Foundry tests green in fork mode (paste run output).
- Package stays TINY (< 5 kB gzipped, zero deps besides viem peer). `npm pack` clean.
- Every semantic claim in the site is sourced (spec link, contract line, or an on-chain
  observation you made — no folklore).

- [ ] Pre/post-adjusted price question answered definitively with on-chain evidence, and the

- [ ] Live multiplier reads on 3+ tokens; fork tests green; type-level tests green.
- [ ] Explainer reads as the best resource on the topic that exists anywhere.
