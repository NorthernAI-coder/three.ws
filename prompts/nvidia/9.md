## The end-to-end truth test

Walk the full loop on real infrastructure and confirm every hop:

1. A real operator runs the node client (02), registers onchain (01), and appears in the
   roster and on the network map (06).
2. A real three.ws job — a forge generation (05) — routes through the router (03) to that
   node under policy `prefer`, runs, and returns a **signature-verified** result (03/07)
   identical in quality to the centralized path.
3. The job **settles onchain** to the node's payout address with a receipt (04); the
   operator sees the earnings + tx in their console (06).
4. A sampled job is **re-run and verified** (07); a fault-injected bad node is caught,
   penalized/slashed, and the user is still served the correct result.
5. With all nodes offline, the same forge/avatar/LLM jobs **fall back to centralized** and
   succeed — the user never regresses (03/05).
6. An external agent uses the **SDK / x402 endpoint** (04/08) to buy compute end-to-end,
   including a federated network leg (08), and gets a verified result.

For each, capture the real evidence (job rows, onchain tx, signatures, logs). Anything you
can't exercise on real infra, say so explicitly — do not paper over it.

## The audit sweep (apply the CLAUDE.md operating rules to all of Phase 4)

- **No mocks / fake data / stubs / TODOs** anywhere in the Phase-4 surface: no simulated
  node, no hardcoded roster, no fake receipt, no `throw new Error("not implemented")`,
  no fallback sample arrays. Grep for them and remove any you find.
- **$THREE / USDC only** across all new code, copy, fixtures, dashboards, SDK, and docs.
  No other token referenced anywhere. Treat any stray reference as a leaked secret.
- **Same job contract** honored across `gcp.js`, `decentralized.js`, and every federation
  adapter — no forked job models or envelope drift.
- **Every state designed** on `run-a-node.html`, the operator dashboard, and the network
  map: loading skeletons, empty (honest "be the next node"), error (actionable), populated,
  overflow. Test at 320/768/1440. Hover/active/focus on every control; ARIA; keyboard nav;
  focus rings; contrast. No console errors or warnings.
- **No errors without solutions:** every failure path (dead node, slow node, bad sig,
  failed settlement, chain hiccup, zero nodes) degrades gracefully with a real fallback.
  Verify the reconciliation cron (04) settles stranded jobs and the router's fallback is
  airtight.
- **Security pass:** node auth is by node key (no central secret as trust root); endpoints
  rate-limited; no node can spoof another's signature, claim a job it can't serve, get
  paid for an unverified result, or escape slashing for a proven bad result. Confirm the
  dispute window and idempotent, no-double-pay settlement hold under retry/race.
- **Metrics honesty:** the "% of traffic served by independent nodes" the network map
  shows (the README north star) is computed from real job accounting (05), not estimated.

## Performance & polish

  SDK are in the bazaar catalog and developer docs; nothing is an orphan page.
- The node operator README and the SDK examples actually work when followed from scratch.
