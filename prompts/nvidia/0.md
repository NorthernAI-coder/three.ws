## The vision

> **Goal (README):** decouple agent inference from any single provider. Anyone
> can run a node; agents pay nodes onchain for compute.

Phases 0–3 built the avatar, the agent, and the onchain identity. But today every
GPU job — face → avatar reconstruction, text → 3D mesh, retexture, talking-avatar
video, LLM inference — runs on **infrastructure we operate** (GCP Cloud Run L4
services, Replicate, Anthropic, a single Livepeer leg). That is the last
centralized dependency in an otherwise ownable stack.

**Phase 4 turns that compute into an open mesh.** Independent operators run a node,
register it onchain, advertise which models their GPU can serve, and get paid per
job in USDC/$THREE with a cryptographic receipt. three.ws's own jobs route to the
mesh first and fall back to the centralized path only when the mesh can't meet the
SLA. Over time, the platform operates *less* of its own GPU fleet, not more.

The loop we are wiring:

> an agent needs compute → the router picks the best healthy node advertising that
> model → the node runs the job and returns a **signed result** → the job settles
> onchain (pay-per-job, cryptographic receipt) → the operator earns → the operator's
> reputation rises → it wins more jobs → repeat.

North star for this phase (from the README verification bar): **≥50% of production
GPU traffic served by independent node operators, at latency parity with the
centralized path.** Every result is signed and verifiable; every payout is onchain
and provable. Decentralized does not mean "best effort" — it means *more* provable,
not less.

This is real infrastructure, not a demo. No simulated nodes, no fake roster, no
mock receipts. A node operator can `docker run` the client on a real GPU box today
and earn real money serving real three.ws jobs.

## What already exists (build ON this — do not rebuild it)

**The compute that gets decentralized — the existing GPU worker fleet:**
- `workers/` — the Cloud Run services that do the heavy work today. Each exposes the
  same HTTP contract: `POST /<verb>` → `202 { task_id, status:"queued" }`,
  `GET /tasks/:id` → `{ status, result_url|result_gcs_url?, error? }`, `GET /health`,
  bearer-auth via a shared secret. Read `workers/README.md` — it is the map.
  - Mesh generation (GPU L4): `model-hunyuan3d`, `model-trellis`, `model-triposr`,
    `model-triposg` (image + scribble), all single-image → textured/geometry mesh.
  - Avatar: `avatar-pipeline-controller` (CPU orchestrator, `POST /reconstruct`),
    `avatar-reconstruction` (InstantMesh), `unirig` (rig/skin/blendshapes).
  - Edit/finish: `texture`, `stylize`, `rembg`, `segment`, `remesh`, `model-text2motion`.
  - Video: `longcat` (talking-avatar MP4).
- `api/_providers/gcp.js` — **the adapter to copy.** It already abstracts every
  worker behind one interface: `submit(request) → { extJobId, eta }`,
  `status(extJobId) → { status, resultGlbUrl?, resultImageUrl?, error? }`. The
  `extJobId` is a base64url JSON envelope carrying the service discriminator + the
  upstream task id. The decentralized provider implements the **same contract** so
  callers don't care where the GPU lives.
- `api/forge.js` + `api/_lib/forge-tiers.js` — how a mesh job is dispatched and
  which engine/tier it routes to. The entry point the router plugs into.
- `api/inference/livepeer.js` — the **early Phase 4 experiment already in the repo**:
  routes LLM inference to Livepeer Studio / the public dream-gateway and compares it
  side-by-side with Claude. The federation prompt (08) generalizes this.

**Payments — the rails per-job settlement rides on:**
- `api/_lib/x402-paid-endpoint.js` — `paidEndpoint(spec)` factory: the 8-step paid
  flow (CORS → method → access → SIWX → 402 challenge → verify → handler → settle).
- `api/_lib/x402-spec.js` — v2 protocol, signing, settlement; `exact` + `upto` schemes.
- `api/x402/*` — 40+ live paid endpoints to match in style.
- Settlement backends already wired: Base via Coinbase CDP, BSC direct, Solana direct.
- `api/_lib/credits.js`, `api/_lib/pricing/catalog.js`, `api/_lib/three-tier.js` —
  pricing/quota/$THREE-gating helpers.

**Schema & infra:**
- `api/_lib/schema.sql` + `api/_lib/migrations/` — add new tables here (timestamped).
- `api/_lib/db.js` (`sql`), `api/_lib/http.js` (`cors`, `wrap`, `method`, `json`,
  `error`, `rateLimited`), `api/_lib/validate.js`, `api/_lib/rate-limit.js`.
- `vercel.json` — register any new route or cron.
- Onchain identity already exists (ERC-8004 on EVM, Metaplex Core on Solana) and the
  reputation/attestation patterns from Phase 3 — reuse them for node registration.

## Global operating rules (every prompt repeats these — non-negotiable)

- Read `CLAUDE.md` and `STRUCTURE.md` first; **CLAUDE.md overrides defaults.**
- **No mocks / fake data / placeholders / TODOs / stubs.** Real nodes, real GPU jobs,
  real signed receipts, real onchain settlement. A simulated node or a hardcoded
  roster is a failed task. If a real second machine isn't available to test against,
  run the node client locally against a CPU-capable job and say so explicitly — never
  fabricate a node.
- **$THREE is the only coin** (CA `FeMbDoX7R1Psc4GEcvJdsbNbZA3bfztcyDCatJVJpump`).
  Node payouts and pricing are denominated in USDC and/or $THREE only. Never reference
  any other token anywhere — code, copy, fixtures, receipts, dashboards, SDK docs. The
  two runtime-data-only exceptions in CLAUDE.md still apply.
- **Same contract as `gcp.js`.** Anything that produces or consumes GPU jobs speaks
  the existing `submit`/`status` envelope and the worker `task_id` shape. Don't fork
  the job model — extend it.
- **Every result is signed; every payout is provable.** No job result is trusted
  without a node signature; no settlement happens without a verifiable receipt.
- **Every state is designed** — loading (skeletons), empty (tells the operator/user
  what to do next), error (actionable + recoverable), populated, overflow. Mobile-first.
- **No errors without solutions.** The mesh degrades gracefully: a dead/slow node never
  fails a user job — the router falls back. Ship the fallback, don't just log the error.
- **Changelog:** every user- or operator-visible change gets a `data/changelog.json`
  entry, then `npm run build:pages`. Internal-only plumbing does not.
- **Tests:** add/extend tests under `tests/`; existing tests still pass (`npm test`).
- Concurrent agents share this worktree — **stage explicit paths only** (never
  `git add -A`/`.`); re-check `git status` and `git diff --staged` before committing.
  Never commit esbuild-bundled `api/*.js` (check `head -1` for `__defProp`).
