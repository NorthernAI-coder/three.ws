## Overall recommendation: 🟡 CONDITIONAL GO — **hold launch until the 4 blockers below clear**

The money-moving core, security posture, on-chain parity, and coin policy are **launch-grade**.
But the full suite is **not all green**, and there are real RED items — one of them on a **live
production endpoint** and one tied to a **same-day database migration**. None are deep
architectural problems; all are closeable quickly. Until they are closed (and the build/test
suite is re-confirmed in a clean CI/Vercel environment — see Blocker 4), this is a **NO-GO**.

When the four blockers are GREEN, this flips to **GO**.

> **Honesty note (per CLAUDE.md "no green-washing"):** this gate was run inside a shared
> multi-agent codespace where (a) other agents were concurrently editing `main`, and (b) the
> local `node_modules` was found **corrupted across ~38 packages** (top-level + transitive
> ESM entry files stripped by an interrupted/concurrent `npm` operation). That corruption
> blocked a fully-green local `vite build`, `check:dist`, and `vitest`/`playwright` run. Where a
> gate could not be run to completion **for that environmental reason**, it is marked
> `BLOCKED (env)` — **not** passed. The authoritative build/test environment is Vercel's clean
> `npm install` (`buildCommand: npm run build:vercel`) plus a clean CI lane (which does not yet
> exist — see Blocker 4 / Prompt 37).

## 7. Rollback plan

| Trigger | Action |
|---|---|
| Bad deploy (errors, broken surface) | Vercel **Instant Rollback** to the previous production deployment (Vercel dashboard → Deployments → Promote previous). Frontend + API revert atomically. |
| Migration / schema regression | Migrations are **additive & idempotent** (`create table/index if not exists`, `add column if not exists`) — forward-safe, so a code rollback does not require a schema rollback. If a destructive change ever ships, restore via Neon **PITR** to the pre-deploy timestamp. |
| Missing-table storm recurs | Re-run `npm run db:bootstrap` (or `scripts/restore-missing-tables.sh --apply` with `OLD_URL`/`NEW_URL`) against the canonical DB; both are idempotent and data-safe. |
| Payment/x402 incident | x402 has an always-on replay guard and server-side amount/recipient/mint verification; disable the affected paid endpoint via its feature gate while investigating — settlement state is in-DB and auditable. |
| App-data corruption | Recover the affected daily snapshot from git history: `scripts/snapshot-export-day.sh <YYYY-MM-DD>`. |
| Code revert hygiene | Use a **neutral** revert message (never echo the reverted title — CLAUDE.md). |
