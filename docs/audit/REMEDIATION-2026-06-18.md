## Investigated ‚Äî already handled (no change)

These audit-pass findings did not reproduce; the code was already correct. Recorded
so they are not "re-found" later.

- **`api/wallet/balances.js` "leaks env var names":** the response is a deliberate,
  structured `503 not_configured` returning the env-var **name** (not value) to help
  self-hosters configure the deploy. This matches the codebase-wide graceful-503
  convention. It exposes config metadata, not a secret ‚Äî left as-is.
- **`api/agent-economy/transact.js` "unvalidated recipient":** the recipient is already
  gated by `if (recipient && isValidPubkey(recipient))` before any transfer. No gap.
- **Modal close buttons missing `aria-label` (compose / go / marketplace):** already
  present (`aria-label="Close ‚Ä¶"`), and `go.html` / `marketplace.html` modals already
  carry `role="dialog"`. Only the two `compose.html` overlays lacked dialog roles
  (fixed above). The remaining audit-pass a11y items were stale.
