# Prompt 02 — Premium venue GLB + named anchors

Build the physical Omniology Arena: a premium, authored 3D venue with named
anchor points (spawn, screen mounts, entry-desk spot, lighting), loaded into the
bootstrap from prompt 01. Model the anchor contract on `/club`'s proven system.

## Read first (required)
- `docs/omniology-arena/README.md`, `docs/omniology-arena/CONTRACTS.md`, `CLAUDE.md`
- `src/club-venue.js` — the named-empty contract. Study `REQUIRED_VENUE_EMPTIES`, `collectVenueEmpties(root, required)` (throws if any missing — **no silent fallback**), and `resolveVenueAnchors(empties, slotCount)`. Note the **underscore naming convention** (`screen_01`, not `screen.01`) and *why* (GLTF `sanitizeNodeName` strips `[]:./`).
- `src/club.js` — how the club loads the venue GLB, runs `collectVenueEmpties`, applies anchors, sets up HDRI via `PMREMGenerator`, lighting rig, bloom/tone-mapping post pipeline, and per-mesh shadow opt-in. Reuse these patterns.
- `scripts/build-club-venue.mjs`, `scripts/build-club-hdri.mjs` — how club assets are generated procedurally with `@gltf-transform/core` and wired to `npm run build:club-venue` / `build:club-hdri`. Mirror this approach.
- `public/club/venue/LICENSES.md` — asset provenance discipline you must follow.
- `src/game/arena/arena.js` from prompt 01 — the `anchors` field you will populate.

## Build
1. **Anchor module** `src/game/arena/arena-venue.js`: export `ARENA_REQUIRED_EMPTIES`, `collectArenaEmpties(root)`, and `resolveArenaAnchors(empties)` returning a typed object:
   ```
   { spawn:{pos,yaw}, screens:[{pos,yaw,width}], desk:{pos,yaw}, lights:[…], camera_intro:{pos,lookAt} }
   ```
   Reuse `collectVenueEmpties` semantics (hard error listing missing anchors). Cover it with a test like `tests/club-venue-load.test.js` → `tests/arena-venue.test.js`.
2. **Venue GLB** at `public/arena/omniology/venue.glb`, generated by a new
   `scripts/build-arena-venue.mjs` (procedural, like the club builder) OR a
   designed GLB if you author one — either way it must contain every
   `ARENA_REQUIRED_EMPTIES` anchor with underscore names. Required anchors at
   minimum: `spawn_01`, `screen_01`…`screen_03` (the contest walls), `desk_01`
   (entry desk), `light_key`, `light_fill`, `light_rim_01..0N`, `camera_intro`.
   Add `npm run build:arena-venue` to `package.json` and document assets in a new
   `public/arena/omniology/LICENSES.md`.
3. **HDRI / environment** `public/arena/omniology/hdri.hdr` via
   `scripts/build-arena-hdri.mjs` (or reuse a suitable existing CC0 asset with
   provenance). Apply via `PMREMGenerator` exactly like club.
4. **Mount in bootstrap**: in `OmniologyArena`, replace the prompt-01 placeholder
   environment — load the venue GLB + HDRI in parallel (with a real progress
   indicator feeding the loading overlay), run `collectArenaEmpties` +
   `resolveArenaAnchors`, and overwrite `this.anchors`. Spawn the local player at
   `spawn.pos/yaw`. Set up the lighting rig from the light anchors. Add a tasteful
   post pipeline (bloom + ACES tone mapping + vignette) gated by a perf check.
5. **Collision / bounds**: define the walkable region (venue footprint) and clamp
   the player to it (client-side clamp like `/play`'s `WORLD_BOUND`; reuse zone
   helpers from `src/game/world-zones.js` if useful). Keep players inside the room.
6. **Performance**: instance/merge repeated structural meshes; one key light with
   shadows, the rest cheap; respect a perf profile (mirror `src/club-perf.js`).
   The venue must hold 60fps on a mid-tier laptop and run on mobile.

## Acceptance criteria
- `/arena/omniology` shows a premium, well-lit, screenshot-worthy room (not a gray
  box). Spawn, lighting, and camera intro come from GLB anchors — confirm by moving
  an anchor in the build script and seeing the runtime follow.
- Missing an anchor throws a clear authoring error (verified by the new test).
- `npm run build:arena-venue` (and the HDRI builder) are reproducible and the
  outputs are committed with a `LICENSES.md`.
- Multiplayer from prompt 01 still works; players spawn at `spawn_01` and are bounded.
- No console errors/warnings. `npm test` passes (including the new anchor test).
- 60fps mid-tier; verified in a real browser.

## Hand-off
`this.anchors.screens[]` and `this.anchors.desk` are the mount points prompts 03
and 04 consume. Do not build screens or the desk here.
