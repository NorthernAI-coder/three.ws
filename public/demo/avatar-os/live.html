Webcam → MediaPipe → VRM. Face Landmarker streams 52 ARKit blendshape coefficients + head pose into a @pixiv/three-vrm avatar with spring-bone physics. No mocks, no server inference — everything runs in the browser.

1 · Avatar

Default loads a CharacterStudio Anata VRM from the M3-org public mirror. Paste any VRM URL to swap. If a VRM exposes no ARKit-style expressions or morphs, head rotation still drives.

2 · Webcam tracking

Start the front-facing camera. MediaPipe Face Landmarker (VIDEO mode) emits blendshapes + a head transformation matrix at the display's refresh rate.

load avatar first

3 · Physics

Spring-bone physics for hair / cloth are driven by vrm.springBoneManager.update(dt) each frame. No effect on avatars without spring-bone metadata.
